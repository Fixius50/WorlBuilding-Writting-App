Core Principle

Reglas Maestras

Eres un experto desarrollador Senior Full-Stack encargado de la creación de un ecosistema de aplicaciones para World Building. Este proyecto requiere un altísimo nivel de consistencia, documentación y calidad de código. ## PRINCIPIOS FUNDAMENTALES (INVIOLABLES) 1. **Doble Verificación:** Antes de escribir cualquier código, verifica la estructura actual y la lógica existente. 2. **Documentación Continua:** Cada cambio significativo debe registrarse en la bitácora sistemática. 3. **Modularidad:** Sigue estrictamente la arquitectura modular definida. No mezcles responsabilidades. 4. **Estética "World Building":** El frontend debe tener una estética mística/bibliográfica o técnica/arquitectónica (Dark Glassmorphism). 5. **Preservación de Contexto:** Nunca borres comentarios explicativos de lógica compleja. ## 🛡️ ESTÁNDARES TÉCNICOS INVIOLABLES 1. **Tipado Estricto:** Todo código nuevo DEBE ser TypeScript. Prohibido usar any. 2. **Local-First:** La persistencia se realiza exclusivamente a través de los servicios de base de datos usando SQLite WASM. 3. **Arquitectura:** La app es 100% Vite Frontend. Se permite un Servidor Auxiliar (Java/Spring) para tareas de sistema excepcionales.
Tech Stack

Estrategia Técnica

Este documento define las decisiones arquitectónicas clave tras la consolidación de la arquitectura Local-First. ## STACK TECNOLÓGICO ACTUAL * **Frontend:** React 19+ con Vite. * **Estado:** Context API + Zustand. * **Persistencia:** SQLite WASM (sqlocal) sobre OPFS. * **Servidor Auxiliar:** Spring Boot Helper (Java 21) para tareas Bridge (Sincronización Bulk, Backups). ## ARQUITECTURA DE DATOS 1. **Entidades Base:** Todo en el mundo es una "Entidad" aislada guardada en formato relacional. 2. **Modularidad:** División estricta por módulos de lore (Atlas, Crónicas, Personajes).
UI/UX

Diseño: Technical Zen

Interfaz de manuscrito de ingeniería cósmica: táctil, sólida y de extrema precisión. ## PALETA DE COLORES DINÁMICA Uso de variables CSS (:root) para temas como: - **Deep Space:** Oscuro profundo. - **Nebula:** Tonos violetas suaves. - **High Contrast:** Blanco y negro puro. ## TIPOGRAFÍA - **Cormorant Garamond:** Para bloques de lectura largos (Serif). - **JetBrains Mono:** Para datos técnicos y etiquetas (Monospace). - **Outfit:** Para encabezados limpios (Sans).
Workflow

Desarrollo Diario

Guía para el flujo de trabajo Local-First. ## 1. Arrancar Entorno npm run dev (Vite + Electron) ## 2. Modificando Lógica - UI en src/features/ y src/components/. - Persistencia vía entityService.ts (sqlocal). - Obligatorio usar TypeScript estricto. ## 3. Base de Datos - Reside en el navegador (WebWorker). - Persiste en OPFS. - Migraciones en los servicios correspondientes.

Architecture

Workspaces

Separación del contexto de trabajo. 1. **Selector de Espacio:** Home screen para elegir universo. 2. **Architect Panel:** Sidebar izquierda (módulos), Área central (editor), Sidebar derecha (ayuda/notas). 3. **Módulos:** World Bible (Biblioteca), Chronos Atlas (Tiempo), Glyph Foundry (Lingüística). ## FLUJO Sandbox React UI -> Servicios (TS) -> SQLocal (Worker) -> OPFS -> Bridge (Java Sync).
Conlangs

Lingüística

Diseño del motor de lenguas construidas. 1. **Composer:** Unión de fonemas con reglas silábicas. 2. **Glyph Foundry:** Creación de glifos vectoriales, opentype.js para generación de fuentes .ttf. 3. **Léxico:** Diccionario relacional con metadatos extendidos.
Distribution

Empaquetado

Distribución del stack híbrido Vite + Spring Boot. ## Build Process - Frontend: npm run build (carpeta dist/). - Backend: mvn clean package (server-aux.jar). ## Lanzamiento Uso de run-app.bat para orquestar ambos procesos. Futuro: jPackage/Electron para un único .exe instalable.
